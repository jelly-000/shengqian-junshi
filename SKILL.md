# 省钱

> 花少钱，也把生活质量稳住。你描述需求，我按「洼地三问法」给你找白菜价平替，并产出一份可执行的省钱方案。

## 这个 Skill 做什么

把一句模糊的省钱需求，变成一份**结构化 Markdown 报告**：先给「直接照做」动作块（去哪个平台、搜什么词、花多少钱、第一步点什么），再简短解释为什么能这么省。让用户不动脑就能开工。

**核心方法（本 skill 的脊柱）——「洼地三问法」**：

> **渠道洼地（去哪买最便宜） ＋ 时段人群（谁在何时甩货） ＋ 精准搜索词（怎么捞） ＝ 把溢价挖掉**

同一件商品，在不同人群、渠道、时段的售价能差出几倍——这是普遍存在的「价格错配」。省钱的本质是找到价格洼地：把货卖白菜价的「特定人群 ＋ 特定时段 ＋ 低价渠道」，用精准搜索捞出来。面对任何一笔花费，填空即可：① 在哪个渠道最便宜？② 是哪一人群、哪一时段在那上面放低价？③ 用什么精准搜索词能捞到？

三类杠杆（与公式互补，作为产出分类视角）：**砍**（Need vs Want）/ **平替**（同等功能更低价）/ **渠道与时机**（大包装·反季·临期·合租·错峰·比价·积分）。

## 何时使用

- 用户想买某物但超预算、想找平替：「想买 iPhone 但只有 3000」「羽绒服太贵有平替吗」
- 用户某类开销过大：「每月外卖花太多」「房租太贵怎么省」
- 用户要性价比方案：「周末想自驾游怎么省钱」「考证资料哪里白嫖」
- 触发词：省钱 / 平替 / 帮我省 / 买不起 / 性价比 / 怎么买划算 / 花太多在 XX / 价格洼地 / 这个好贵 / 有便宜的没 / 太贵了
- 也可由用户直接描述需求自然触发（如「想买 iPhone 但只有 3000」「每月外卖花太多」）
- 在任意支持 Skills 的 Agent 中，也可显式调用：`/省钱` 或 `@省钱`（视 Agent 支持的命令语法）

## 类目覆盖

吃、穿、住、行、用（数码/家电）、玩（娱乐）、学（自我提升）、金融/理财、日用/订阅审计、健康（仅安全项）。策展细节见 `references/pingti-kb.md`。

## 工作流（严格按顺序）

### Step 1 · 接收需求
- 若消息已含主题+场景，跳过提问；否则最多问 **1–2 个**关键澄清（预算上限 / 使用场景 / 地域），其余给默认。
- 不要闷头做太久，问完关键项就动手，让报告说话。

### Step 2 · 第一性原理拆解
区分「要的东西」和「要的结果」：用户真正要的体验/结果是什么？（例：「买 iPhone」→ 要的是「能流畅拍视频+社交够用」，未必是苹果新款。）

### Step 3 · 套用公式（核心）
对该需求按三栏填空，从 `references/pingti-kb.md` 取常规范例：
- ① **低价平台**：该物在哪个低价平台最便宜？
- ② **特定人群段**：是哪一人群、哪一时段在那上面放低价？
- ③ **精准搜索**：用什么感觉词/小众词能捞到？
- 同步叠加三杠杆（砍 / 平替 / 渠道与时机），给出 2–4 个候选方案。

### Step 4 · 实时搜 + 知识库自更新（按需）
对需要具体商品/实时价格的需求，用 Agent 的**联网搜索能力**（WebSearch / WebFetch 等）补全：query 带当前年份等时效词；标来源与日期；结果标注「价格以实时搜索为准」。

**知识库自更新（闭环）**：生成报告时，对本需求所属类目，用 Agent 的联网搜索能力（WebSearch / WebFetch 等）抽查 1 次该类目最新价/平台规则；若与 `references/pingti-kb.md` 中该条的记忆偏离 >20%，将新价/新规则与今日日期（`更新于 YYYY-MM`）回流写入该条，使 KB 不靠「实时搜索为准」口头免责而持续保鲜。

### Step 5 · 生成结构化报告
报告必须以「直接照做」动作块开头，用户未读任何解释即可照做；解释性章节为压缩后的背景，不堆砌。
按下方模板写入 `<当前工作目录>/省钱方案-<主题>.md`，并展示给用户（WorkBuddy 中可用 present_files）。

**省钱账本（跨会话累计）**：生成报告前读 `~/.shengqian-junshi/账本.md`（文件不存在视为空），按「主题」去重累加得历史累计（同一主题只计一次，重跑时更新该行而非新增，避免重复累加）；报告中「累计省 ¥Y」= 各主题「本次省」之和（注意各案例周期可能不同，不强行相加成单一数字）。报告写完后向该账本写入一行——若「主题」已存在则**更新**该行金额，否则**追加**——格式：`| <YYYY-MM-DD> | <主题> | <对照基准：省的是相对谁> | <本次省金额+周期> |`（append-only 单文件，无数据库；首次运行自动创建）。**对照基准必须为用户真实会考虑的次优方案，禁止拿专柜 / Pro 等虚高价格当基准。**

### Step 6 · 自检（交付前必做）
- 报告开头必须是「直接照做」动作块（含平台+精准搜索词+预期价+第一步具体可执行），用户未读任何解释即可照做；
- 解释章节（你要的是啥/为什么能省/平替对照）须精简，不堆砌；
- 平替对照表列数压缩（方案/去哪/搜什么/预期价/省/折损），金额自洽；
- 附**避坑提示**（≤3 条关键项）与统一**免责页脚**；链接可点击；
- **省钱账本**「本次省」须有真实对照基准（非虚高专柜 / Pro 价）、账本按主题去重（重跑不重复累加）、且已写入一行进 `~/.shengqian-junshi/账本.md`；
- **不含已剔除的违规/伤身项**；具体价格不写死，标「实时搜索为准」。

   - 链接模板（中文电商务必用 **GBK 编码**的搜索词，这些平台按 GBK 解析搜索参数，UTF-8 会乱码/打不开）：拼多多 `https://mobile.yangkeduo.com/search_result.html?search_key=<GBK编码词>`；闲鱼 `https://www.goofish.com/search?q=<GBK编码词>`；1688 `https://s.1688.com/selloffer/offer_search.htm?keywords=<GBK编码词>`。

2. （备选）打开 **<平台B>**，点链接 **<平台B搜索直链>**（搜词「**<精准搜索词2>**」），预期 <价格>。

## 内容安全与免责（硬约束）

- **删除清单（绝不进报告）**：教唆违法（代购/自用违禁药）、无处方医疗建议（具体药品当万能药）、退货滥用等违规薅羊毛、伤身/欺诈性省钱法。
- 健康/医疗内容仅作思路提示，**非医疗建议，遵医嘱**；金融内容**非投资建议**。
- 价格一律标「以实时搜索为准」，不承诺写死数字、不保证绝对最低。
- 报告统一附免责页脚。
